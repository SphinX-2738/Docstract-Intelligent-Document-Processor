DOCSTRACT
Intelligent Document Processor — Gen AI Portfolio Project

What it does

Upload any invoice, resume, or email in plain text and Docstract extracts all relevant fields, validates the output, scores confidence per field, and flags anything uncertain for human review — all in a single API call.

How it works

Four steps run on every document:

01
Extract — document text is sent to an LLM with a document-type-aware prompt and a strict Pydantic schema to fill
02
Validate — extracted data passes through business logic checks (is the total > 0? is the email body present?)
03
Confidence Score — every field gets scored 0–100%. Low confidence fields are flagged for human review instead of silently passing bad data
04
Cost Track — real token usage captured per document with cost calculated across 9 providers

Tech Stack

Python FastAPI Groq API LLaMA 3.3 70B Pydantic ChromaDB Sentence Transformers

Key Features

▸Supports invoices, resumes, and emails out of the box
▸Semantic search via ChromaDB handles documents of any size
▸Provider-agnostic cost tracking — switch models with one line
▸REST API with auto-generated interactive docs at /docs
▸Human-in-the-loop flagging for uncertain extractions

Why I Built This

Most AI projects extract data and hope it's correct. Docstract extracts, validates, scores, and proves it — because shipping reliable AI is harder and more valuable than shipping fast AI.

How to use

01 Paste your document text into the input box, or drag and drop a .txt file — or load one of the sample documents to try it instantly
02 Select the document type — Invoice, Resume, or Email — using the tabs below the input
03 Hit EXTRACT DOCUMENT or press Ctrl + Enter to run the extraction
04 Review the extracted fields, confidence scores, and token usage in the results panels below — click any field to copy its value
